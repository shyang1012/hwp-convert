# `/배포` — dev→main e2e + PR + npm publish

npm 라이브러리 배포 절차. AGENTS.md §배포 워크플로우 정합. STEP 순차 실행.

## 주의

- `git push --force` / `git reset --hard` 금지.
- uncommitted 변경 있으면 중단(먼저 `/push`).
- 테스트 실패 시 배포 금지.
- `.npmrc`(npm 토큰)·시크릿 출력 노출 금지.
- npm publish 는 PM 승인 하 실행. **2FA 우회 Automation 토큰 필요**.

## STEP 0 — 사전 확인

```bash
git status --short          # 비어있어야 함. 아니면 중단 → /push 먼저
git branch --show-current   # dev 여야 정상
```
- 미커밋 있으면: `[STEP 0] ❌ 미커밋 변경 — /push 먼저` 후 중단.

## STEP 1 — E2E Smoke PreFlight (배포 차단 게이트)

이번 배포 변경이 아래 trigger 영역에 영향을 주면, 대표 경로(critical path) e2e 를 **신설/갱신**했는지 점검.

**Trigger 영역**:
1. OWPML 생성(hwpxBuilder/writer/owpml) — mimetype·속성·secPr·표·이미지 구조
2. 변환 진입(mdReader/htmlReader/converter) — md/html → IR
3. 읽기(hwpxReader) — extract/round-trip
4. 공개 API/CLI(index/cli) — 시그니처·옵션

```bash
git fetch origin                                  # 로컬 main 이 stale 일 수 있음 → origin 기준 비교
git diff --name-only origin/main..dev -- src/ | head -30
```
- 변경 있음 + `test/e2e.test.ts`(또는 owpml-compat 등 full 파이프라인 e2e) 신설/갱신 → `[STEP 1] ✅ trigger 변경 + e2e 갱신 확인`
- 변경 있음 + e2e 미갱신 → 🔴 `[STEP 1] 🔴 trigger 변경 but e2e 미갱신 — 대표 경로 e2e 신설 후 재진입 (계속? y/N)` PM `y` 아니면 중단.

## STEP 2 — 전체 검증

```bash
npm run build && npm test    # 109+ 전부 통과해야
```
- 실패 시 `[STEP 2] ❌ 테스트 실패 — 배포 중단`.
- 가능 시 변환 산출물 한글 열림 수동 확인(hwp MCP `hwp_open_document` + `hwp_get_tables`).

## STEP 3 — 버전 bump (dev에서 먼저) + dev → main PR

🔴 **버전 bump 는 머지 전 dev 에서 한다.** main 은 보호(PR 필수 + `enforce_admins`)라 버전 커밋을 직접 push 할 수 없다. dev 에서 bump 하면 PR 이 버전을 main 으로 실어 나르고, main 엔 STEP 4 에서 **태그만** 올리면 된다(태그 push 는 분기 보호와 무관). → main package.json 항상 정합, 드리프트·backsync 충돌 0.

```bash
npm version <patch|minor|major> --no-git-tag-version   # package.json 만 bump (태그는 STEP 4에서 main 커밋에)
git commit -am "release: v<버전> — <요약>"
git push origin dev
gh pr create --base main --head dev \
  --title "release: v<버전> — <요약>" \
  --body "변경 요약 / 검증 결과(테스트 N개) / e2e 범위 / 한글 열림 확인"
```
- PR URL 출력. PM 검토·머지 대기(또는 PM 명시 시 `gh pr merge <N> --squash`).

## STEP 4 — main 태그 push → GitHub Actions 자동 배포 (PR 머지 후)

```bash
git fetch origin
git tag v<버전> origin/main          # 머지된 main 커밋(=bump된 package.json)에 태그
git push origin v<버전>               # 태그 push → .github/workflows/publish.yml trigger
```
- 🔴 main 브랜치에 **커밋을 push 하지 않는다** — 태그만 올린다(보호 분기 우회 불필요).
- **npm publish 는 GitHub Actions(OIDC Trusted Publisher)가 자동 수행** — 수동 `npm publish` 불필요(로컬 토큰도 불필요).
- v 태그 push 시 `publish.yml` 이 OIDC 로 토큰 없이 publish(provenance 포함). prepublishOnly(clean→build→test) 자동 실행.
- 진행/결과 확인: `gh run list --workflow=publish.yml --limit=1` → `gh run watch <id>`. 성공 후 `npm view hwp-convert version`.
- (Actions 실패·긴급 시 fallback) 로컬 수동: `npm publish --access public` (Automation 토큰 필요).

## STEP 5 — 마무리

dev 가 이미 bump 된 버전을 가지므로 **backsync 불필요**(과거 main→dev 머지 충돌 원인 제거). dev 는 squash 머지로 main 보다 앞서 있는 정상 상태.

```bash
npm view hwp-convert version         # 게시 버전 == package.json 버전 확인
```

완료 출력:
```
[배포 완료]
- PR: {url} (머지 {해시})
- npm: hwp-convert@{버전} 게시 (tag v{버전})
- 검증: 테스트 {N}개 + e2e {범위}
```
