# `/배포` — dev→main e2e + PR + npm publish

npm 라이브러리 배포 절차. AGENTS.md §배포 워크플로우 정합. STEP 순차 실행.

## 주의

- `git push --force` / `git reset --hard` 금지.
- uncommitted 변경 있으면 중단(먼저 `/push`).
- 테스트 실패 시 배포 금지.
- `.npmrc`(npm 토큰)·시크릿 출력 노출 금지.
- npm publish 는 PM 승인 하 실행. **2FA 우회 Automation 토큰 필요**.

## STEP 0 — 사전 확인

```bash
git status --short          # 비어있어야 함. 아니면 중단 → /push 먼저
git branch --show-current   # dev 여야 정상
```
- 미커밋 있으면: `[STEP 0] ❌ 미커밋 변경 — /push 먼저` 후 중단.

## STEP 1 — E2E Smoke PreFlight (배포 차단 게이트)

이번 배포 변경이 아래 trigger 영역에 영향을 주면, 대표 경로(critical path) e2e 를 **신설/갱신**했는지 점검.

**Trigger 영역**:
1. OWPML 생성(hwpxBuilder/writer/owpml) — mimetype·속성·secPr·표·이미지 구조
2. 변환 진입(mdReader/htmlReader/converter) — md/html → IR
3. 읽기(hwpxReader) — extract/round-trip
4. 공개 API/CLI(index/cli) — 시그니처·옵션

```bash
git diff --name-only main..dev -- src/ | head -30
```
- 변경 있음 + `test/e2e.test.ts`(또는 owpml-compat) 신설/갱신 → `[STEP 1] ✅ trigger 변경 + e2e 갱신 확인`
- 변경 있음 + e2e 미갱신 → 🔴 `[STEP 1] 🔴 trigger 변경 but e2e 미갱신 — 대표 경로 e2e 신설 후 재진입 (계속? y/N)` PM `y` 아니면 중단.

## STEP 2 — 전체 검증

```bash
npm run build && npm test    # 109+ 전부 통과해야
```
- 실패 시 `[STEP 2] ❌ 테스트 실패 — 배포 중단`.
- 가능 시 변환 산출물 한글 열림 수동 확인(hwp MCP `hwp_open_document` + `hwp_get_tables`).

## STEP 3 — dev → main PR

```bash
git push origin dev          # dev 최신화
gh pr create --base main --head dev \
  --title "release: <요약>" \
  --body "변경 요약 / 검증 결과(테스트 N개) / e2e 범위 / 한글 열림 확인"
```
- PR URL 출력. PM 검토·머지 대기(또는 PM 명시 시 `gh pr merge --squash`).

## STEP 4 — 버전 bump + npm publish (main 머지 후)

```bash
git checkout main && git pull origin main
npm version <patch|minor|major>     # package.json + git tag
git push origin main --tags
npm publish --access public          # prepublishOnly: clean→build→test 자동
```
- E403(2FA) 시: Classic Automation 토큰 필요 → `npm config set "//registry.npmjs.org/:_authToken=<토큰>"` (PM 직접).
- 성공 후 `npm view hwp-convert version` 으로 확인.

## STEP 5 — 마무리

```bash
git checkout dev && git merge main   # main 의 version bump 를 dev 에 backsync
git push origin dev
```

완료 출력:
```
[배포 완료]
- PR: {url} (머지 {해시})
- npm: hwp-convert@{버전} 게시
- 검증: 테스트 {N}개 + e2e {범위}
```
