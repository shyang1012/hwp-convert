# `/시작` — 세션 컨텍스트 로드 + 상태 보고

프로젝트 컨텍스트를 로드하고 현재 작업 상태를 보고하라.

## STEP 0 — 메모리 로드 (bd, 3단 포인터)

메모리는 bd 단일 원장(상세 `.claude/MEMORY.md`). 본문 일괄주입 없이 인덱스→핀포인트 recall.

```bash
bd prime                       # 골격(~1.7KB 고정, 메모리 본문 미주입)
bd memories                    # 인덱스(키 + 한 줄 요약) — 전체 한눈에
```
- 인덱스에서 **`session-*` 키 최근 3개만** 본문 펼침: `bd recall session-<날짜>` (3건).
- 그 외 키는 인덱스만 보고, 작업에 필요할 때 해당 키를 `bd recall` 로 조회.
- bd 미초기화 시에만 건너뜀(실제 실행 결과로 판단).

## STEP 1 — 운영 정책 상기

`AGENTS.md` 를 읽어 핵심 정책을 상기한다: dev 개발 / main 배포(e2e+PR) / beads 이슈 / local-first / 보안(.npmrc·.claudeignore). 아키텍처 상세는 `CLAUDE.md`.

## STEP 2 — 브랜치 + 최근 이력

```bash
git branch --show-current        # dev 정상
git log --oneline -5
git status --short                # 미커밋 잔여 확인
```

## STEP 3 — beads 작업 현황

즉시 착수 가능한 작업을 **실제로 실행**해 확인:

```bash
bd ready
```
- 정상 출력(목록 또는 "0 issues")이면 보고에 포함.
- 실행 실패 메시지가 확인된 경우에만 건너뜀. 🔴 미설치·미초기화 가정 금지(실제 실행 결과로만 판단).

## STEP 4 — 상태 보고

```
[세션 시작] 컨텍스트 로드 완료
- 메모리: 인덱스 {N}건 / 최근 세션 {최대 3}건 로드
- 브랜치: {현재} / 미커밋: {N건 또는 없음}
- 최근 작업: {git log 최근 1~2개}
- 착수 가능(bd ready): {Top 3 또는 0건}
- 메모: {미배포 변경·진행 중 작업이 있으면 한 줄}
```
