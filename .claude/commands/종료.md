# `/종료` — 세션 마무리 (beads 동기화 + 커밋)

세션 종료 전 local-first 체크리스트를 수행하라.

## STEP 1 — 세션 요약 (PM 확인)

이번 세션의 **완료된 작업 / 결정사항 / 발생한 이슈 / 다음 할 일**을 추출해 보여주고 승인받는다.

```
📋 세션 요약
### 완료된 작업
- ...
### 결정사항 (있으면)
- ...
### 발생한 이슈 (있으면)
- ...
### 다음 할 일
- ...
이대로 저장할까요?
```

## STEP 2 — beads 동기화

(bd 미초기화 시 건너뜀)

```bash
bd close <id>                              # 완료된 작업 이슈 닫기
bd create --type <type> --title "<제목>"   # 새 다음 할 일 등재
bd note <id> "<진행 메모>"                 # 진행 중 핸드오프 로그
```
- 지속 정책/결정은 `--type decision` (정책 유효한 동안 OPEN).

## STEP 3 — 품질 게이트 (코드 변경 시)

```bash
npm test          # 통과 확인
```
- 실패 잔존 시 요약에 명시 + 다음 할 일로 등재.

## STEP 4 — 로컬 커밋 + 옵션 push

```bash
git status --short
git add -A        # .npmrc·*.key·*.env 제외
git commit -m "<prefix>: <요약>"
```
- 백업 원하면(또는 PM 요청): `git push origin dev` (작업 브랜치만).

## CRITICAL

- 명시 배포 요청 없이 `main` push/머지 금지(배포는 `/배포`).
- 명시 요청 없이 `bd dolt push` / force-push / 브랜치 rewrite 금지.
- 무관한 로컬 변경 보존.

## 완료 메시지

```
✅ 세션 종료 완료
- beads: close {N}건 / 신규 {M}건
- 커밋: {해시} — {메시지} (push: {dev / 로컬만})
- 다음: {다음 할 일 1~2개}
수고하셨습니다! 🎉
```
